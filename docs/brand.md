## Core Positioning

Verity is an AI software delivery framework that turns ideas into verified production releases.

Unlike AI coding tools that stop when code is written, Verity treats shipping as the definition of done. It uses specialized AI roles to design, build, review, release, and verify software through a GitHub-native production pipeline.

## Messaging Pillars

### 1. Shipping is the spine

Most AI coding tools stop when code is generated. Verity keeps going until the change is tested, deployed, and verified.

**Key phrase:** Code is not the finish line.

### 2. Specialized roles, clear contracts

Verity breaks work into disciplined roles: architect, builder, reviewer, release operator, verifier, and more. Each role hands work off through explicit contracts instead of loose agent memory.

**Key phrase:** Built like a team, not a chatbot.

### 3. GitHub is the source of truth

Progress is derived from real engineering artifacts: pull requests, commits, checks, releases, and deployments.

**Key phrase:** The repo tells the truth.

### 4. Review is adversarial

Verity does not just produce code. It challenges its own work before release.

**Key phrase:** Every change has to earn green.

### 5. Done means verified

Verity validates the running app the way a user would: by interacting with the real interface and confirming the feature works.

**Key phrase:** Done means deployed and proven.

## Product Architecture / Brand System

### Verity Relay

**The agent handoff and orchestration engine.**

Relay coordinates specialized roles across the delivery lifecycle. It moves work from architect to builder to reviewer to release operator to verifier through clear contracts and visible state transitions.

**Positioning line:** Role-based AI delivery orchestration for real software projects.

### Verity Shipyard

**The build, deploy, and release environment.**

Shipyard is where code becomes a release. It handles build pipelines, test execution, deployment steps, release packaging, and operational readiness.

**Positioning line:** Where AI-built software becomes production-ready.

### Verity Ledger

**The GitHub-derived source-of-truth layer.**

Ledger reads the real project state from GitHub activity: merged work, releases, deployments, checks, and issues. It avoids stale task files and unreliable agent memory.

**Positioning line:** Project truth from real engineering records.

### Verity Gate

**The review, security, and checkpoint system.**

Gate enforces quality before a change moves forward. It includes code review, test gates, security checks, policy checks, and release readiness criteria.

**Positioning line:** Every change must pass before it ships.

### Verity Verify

**The live app verification layer.**

Verify confirms that the deployed application actually works from the user’s point of view. It can click the button, inspect the result, and prove that the feature works in the running environment.

**Positioning line:** Verification against the real app, not just the codebase.

## Suggested Workflow Narrative

1. **Describe** what you want built.
2. **Architect** converts the request into a technical plan.
3. **Relay** routes the work through specialized roles.
4. **Builder** implements isolated changes.
5. **Gate** reviews, tests, and challenges the work.
6. **Shipyard** builds, releases, and deploys.
7. **Verify** confirms the running app works.
8. **Ledger** records the true state from GitHub and deployments.
